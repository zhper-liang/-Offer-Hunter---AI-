## Requirements Summary

### Problem Statement

当前项目使用自定义 ReAct agent 实现，需要迁移到 LangChain 生态以获得更好的工具链、社区支持和可维护性。同时进行现代化重写，利用 Python 3.12+ 和 FastAPI 新版本特性。

### Scope

**In Scope:**
- 迁移 AI Agent 系统到 LangGraph
- 迁移 LLM Provider 到 LangChain ChatModels
- 迁移工具系统到 LangChain Tools
- 迁移 RAG Pipeline 到 LangChain 集成
- 迁移 Session 管理到 LangGraph Checkpoint
- 升级 Python 和框架依赖
- 保持所有现有功能

**Out of Scope:**
- 前端框架迁移（保持 React 18）
- 新增功能（仅迁移现有功能）
- 数据库迁移（保持文件存储）

### Success Criteria

- [ ] 所有现有功能正常工作
- [ ] 自动化测试通过率 > 95%
- [ ] 性能指标不低于迁移前（响应时间 ±10%，内存使用 ±20%）
- [ ] 代码量减少 30%+
- [ ] 架构复杂度降低（减少自定义抽象层）

### Constraints

- 保持 API 接口兼容性（前端无需大改）
- 支持所有现有 LLM provider（包括国内 provider）
- 保持语义分块器的自定义能力
- 保持 Skill 系统的行为叠加能力

### Assumptions

- 用户希望一次性完成完整功能迁移
- 用户接受全面验证策略（测试+功能+性能）
- Python 3.14.4 与 LangChain 兼容
- 国内 LLM provider 可通过 ChatOpenAI + base_url 适配

## Architecture

### Approach

采用 **LangGraph 状态图** 管理 agent 流程，替代自定义 ReAct 循环。使用 LangChain 统一 LLM 接口，减少自定义抽象层。

### Key Components

- **LangGraph StateGraph**: 替代自定义 ReAct 循环，提供可视化状态管理
- **LangChain ChatModels**: 替代自定义 LLMProvider，统一 LLM 接口
- **LangChain Tools**: 替代自定义 BaseTool，自动参数验证
- **LangChain VectorStore**: 保留 ChromaDB，使用 LangChain 集成
- **LangGraph Checkpoint**: 替代自定义 SessionManager，支持持久化

### Data Flow

```
用户输入
    ↓
LangGraph StateGraph
    ↓
感知节点 (LLM 调用)
    ↓
工具执行节点 (并行执行)
    ↓
目标检查节点 (是否完成?)
    ↓ (否)
回到感知节点
    ↓ (是)
输出结果
```

## Technical Considerations

- **Python 3.14.4 兼容性**: 需要提前验证 LangChain 是否完全支持，准备降级方案
- **国内 LLM provider**: 使用 ChatOpenAI + 自定义 base_url 适配，保留自定义 provider 作为备选
- **语义分块器**: 保留自定义实现，但集成到 LangChain 文档加载器
- **Skill 系统**: 在 LangGraph 节点中实现 Skill 匹配逻辑，保持行为叠加能力
- **流式输出**: 使用 LangGraph 的流式能力，保持 SSE 兼容性
